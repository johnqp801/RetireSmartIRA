## The six Alan-feedback fixes (all on `main`)
- **#1 keyboard trap** (prior session) — reusable `.dismissableKeyboard()`, iOS-gated, macOS no-op. `KeyboardDismiss.swift`.
- **#3 gray heat-map** (prior session) — `SSCouplesMatrixColor` 3-color scale normalized across matrix min/max; degenerate min==max returns tealRamp3 (no divide-by-zero). Verified in final review (was salvaged un-reviewed).
- **#2 blank couples table** — ROOT CAUSE: the one-claimed strip filtered `ssCouplesMatrix()` by the claimed spouse's `plannedClaimingAge`, but `couplesMatrix` clamps each spouse's min age to their CURRENT age, so anyone who claimed a year+ ago (or `isAlreadyClaiming`) has a locked age not in the matrix -> empty -> blank. FIX: new pure engine `SSCalculationEngine.couplesStrip(...)` holds the claimed spouse at their TRUE locked age and varies the deciding spouse, reusing the exact per-cell math. New `DataManager.ssCouplesStrip()` wrapper. **Fixed in BOTH `SSCouplesStrategyView` AND the primary-path `SocialSecurityPlannerView.couplesStripBestCell`** (the top-level SS tab, the blank Alan most likely saw — the implementer originally flagged it out-of-scope; controller folded it in) + empty-states in both. `SSCouplesStripTests.swift`.
- **#4 SS chart colors too similar** — the cumulative chart drew all 9 claim-age lines in a colliding single-hue teal ramp. DECISION (John): "Show key ages only" — plot 62 / FRA / 70 + the user's planned age (highlighted), each a distinct locked-palette token, via a pure `SSCumulativeChartColors.displayLines` that chart+colorScale+legend all derive from (no drift). Wired up the previously-dead `highlightClaimingAge`. `SSCumulativeChartColorsTests.swift`.
- **#5a SS withholding -> IRS W-4V rate picker** — new `SSWithholdingRate` enum (none/7/10/12/22%), `IncomeSource.ssWithholdingRate`, `effectiveFederalWithholding` resolves rate x annualAmount at the consumption point; engine dollar semantics unchanged. DECISION: **LAZY back-compat** (not the plan's eager-snap) — legacy SS keeps EXACT dollars until the user edits, so 0 silent data change and 0 test rebaselines. `SSWithholdingRateTests.swift`.
- **#5b non-SS $/% withholding toggle** — `FederalWithholdingMode` (nil==legacy .dollars), `federalWithholdingPercent`; `effectiveFederalWithholding` extended; SS branch leak-proof. New non-SS default .percent, legacy opens .dollars byte-identical. `IncomeWithholdingModeTests.swift`.
