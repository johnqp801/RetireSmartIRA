## 1. Tim Lucas — the feedback that started it

**Email 1 (4 questions):** (a) add a brokerage account (15% LTCG)? (b) can the app model IRA distributions/drawdowns? the model only grows IRAs; (c) balances look too large because there are no drawdowns; (d) can't find the Tax Summary PDF.

**Audit answers (verified in code):**
- **PDF:** exists. Mac, Tax Summary tab, **top of the left column under the "Tax Year" card**, button **"Export PDF for CPA"** (`doc.richtext`); opens a Save dialog → `TaxSummary_[name]_[year].pdf` ([DashboardView.swift:120-133](RetireSmartIRA/DashboardView.swift:120), wide layout [:84-89](RetireSmartIRA/DashboardView.swift:84)). The label is why he missed it.
- **Drawdown:** the RMD Calculator tab already has a year-by-year **RMD** projection (grow at rate, subtract prior RMD) — `projectBalance` [RMDCalculatorView.swift:1449](RetireSmartIRA/RMDCalculatorView.swift:1449), table [:1219](RetireSmartIRA/RMDCalculatorView.swift:1219), 5/10/15/20 picker [:1031](RetireSmartIRA/RMDCalculatorView.swift:1031). But it starts at RMD age (73/75) and models no **voluntary pre-RMD** drawdown — exactly Tim's gap.
- **Brokerage:** no taxable-account type ([AccountModels.swift:51](RetireSmartIRA/AccountModels.swift:51) = Trad/Roth IRA + 401k + inherited only). Cap-gains/qual-div exist as income inputs ([IncomeModels.swift:118](RetireSmartIRA/IncomeModels.swift:118)) but no brokerage account that holds/grows/draws.

**Worked example computed for John** (62yo, $1.5M, 5%, RMD age 75): balance grows untouched 13 yrs to ~$2.83M at 75; RMDs (4.1% rising) stay below 5% growth so the balance keeps climbing, peaks ~$2.88M at age 80, only then declines. 20-yr window stops at 81 — never shows the depletion. This proved Tim's point and motivated the 40-yr horizon.

**Email 2 — Tim's requirements (the V2 user stories):**
1. Optimize Roth conversions over the bridge years, **explicitly NOT optimizing for max longevity value** (objective must be user-selectable).
2. Model **pro-rata** vs **sequential** drawdown across IRAs/brokerage/Roth (Roth last).
3. Draw to meet **budget**; guaranteed income tapped first, IRA fills the gap; 4–4.7% (Bengen) as a target.
4. **Inflate everything**: balances, IRMAA thresholds, standard deduction, brackets.
5. Definitely **optimize withdrawal order**; brokerage first early, "more nuanced" now.

**Market research:** decumulation is the paid/premium feature across the market (Boldin $144/yr 250k+ users; ProjectionLab premium Roth optimizer). Closest competitor **RetIQ** — native iOS (NOT native Mac; iPad-on-Mac), on-device, **$69.99 one-time**, already ships tax-aware drawdown + 5 buckets + Roth optimizer; shipped v3.0 2026-06-01; **no traction yet** (no ratings). RetireSmartIRA's edges: real native macOS, audited tax/state/IRMAA/ACA accuracy, polish, content/SEO. Don't chase RetIQ's breadth; win on native Mac + accuracy + focused bridge-years drawdown.

## 4. Decisions made

**Version/scope (decision-log 2026-06-18):** No standalone 1.8.8. **V1.9 = contained drawdown feature + NJ phaseout, one App Store submission, one Apple review** (John: "I don't want to wait for 2 apple reviews"). The "1.9" label is reassigned (healthcare bundle already shipped). 2.0 stays the big decumulation + multi-year engine.

**Product principle (decision-log 2026-06-18):** single-year **Scenarios** and **Tax Summary** are CORE and must NOT be replaced/removed by 2.0/2.1. Multi-year capability is **additive — likely a brand-new "Multi-Year Plan" tab**. Verify augment-not-supplant during the 2.0 reconciliation.

**V1.9 drawdown design (the 9 brainstorm decisions, via visual companion):**
1. Input modes: **both B (spending gap) and C (withdrawal rate)**, toggle.
2. Mode B guaranteed income: **time-varying**, pulled from SS tab (claiming age + amount) + Income & Deductions (pension), at **gross/face value**.
3. Chart: **A+** — clean balance line default, cash-source bars expandable.
4. Inflation: **single rate** (default 2.5%), separate from growth; inflates spending + guaranteed income + IRMAA threshold; **nominal default + today's-dollars toggle**.
5. IRMAA overlay: **Medium** — projected-income line + inflated tier-1 flag, labeled approximate (Lite=none, Full=2.0).
6. Accounts: **Traditional IRA + 401(k) aggregate only** (Roth/brokerage = 2.0/2.1).
7. Household: **pro-rata** split; each spouse's RMD at their own age.
8. Legacy: **(b) propagate drawdowns into the Legacy/estate projection** (the one intentional cross-tab dependency).
9. Architecture: **standalone tiny projector on current main**, not the 2.0 engine.

**Placement:** new "Retirement Drawdown" section in the RMD Calculator tab, below the existing RMD Projections table; horizon picker extended to 40 yrs. Verified isolation: Scenarios/Tax Summary use current-year `calculatePrimaryRMD()`, never `projectBalance`/`growthRate`.
