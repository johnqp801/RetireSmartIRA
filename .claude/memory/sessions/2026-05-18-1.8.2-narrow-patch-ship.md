## Today's debugging arc ŌĆö what we learned

1. **Memoization didn't fix the hang.** Build 42 added `EngineMemoCache` for 8 hot properties; cut hang from 3.29s ŌåÆ 2.27s on iPad (~30% improvement). Real but insufficient. The remaining cost isn't in the 8 memoized properties.

2. **The hang is NOT in the new Phase 1-3 view mounts.** Bisect branch with all 7 new mounts disabled still hung at the same call stack (`scenarioSummaryCard.getter` ŌåÆ `wideRightGroupA` ŌåÆ `wideBody` ŌåÆ `body`). So the cost is in pre-existing TaxPlanningView code that's somehow getting triggered too many times during tab transition.

3. **Cost is heavily AppKit-layout-driven.** Time Profiler trace showed ~520ms in `CA::Transaction::commit` + `run_commit_handlers` (SwiftUI/AppKit committing layout). Only ~80ms in actual RetireSmartIRA code per body call. But the body gets re-invoked dozens of times during the tab transition ŌĆö cumulative cost spikes.

4. **Apple version-string surprise.** `MARKETING_VERSION = 1.8.1.1` is rejected at upload time (error 90060): "must be a period-separated list of at most three non-negative integers." We were caught off-guard because the project's existing pattern uses 3 segments. Renamed to `1.8.2`.
