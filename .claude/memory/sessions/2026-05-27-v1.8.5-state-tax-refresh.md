# Session — v1.8.5 state tax refresh (full)

**Date:** 2026-05-27
**Branch:** `1.8.5/state-tax-refresh` (worktree)
**Outcome:** Ready to ship

## What we did

### Backstory
v1.8.4 shipped with state tax brackets in mixed vintages (TY 2023-2025).
A tester screenshot showed CA TY 2023 brackets in the UI. User instructed
"we need everything in the end to be 100% correct for tax year 2026" and
chose **Path 1**: TY 2026 actuals where published, latest TY 2025 elsewhere,
all backed by state DOR / official forms / enacted legislation as primary
source (TriSTAR source #1 mandatory).

### Tier 1-3 (prior chapters, shipped in this branch before Bucket 2)
16 states materially refreshed:
- **Tier 1** (single-liners): MS, MI, ID, AR
- **Tier 2** (structural touch): CT, WV, HI, MD
- **Tier 3** (full restructures): MA, LA, KS, MT, ND, SC, OH + CA (the bug
  that started it all)

All achieved full 5-of-5 TriSTAR validation.

### Bucket 2 (this chapter)
Closed the gap on the 13 acceptably-stale states + Nebraska. After research:
- **10 needed code changes:** DE, ME, MN, MO, NE, NY, OK, RI, VT, WI
- **4 verified already current:** KY, IN, UT, DC

Key structural changes:
- **Nebraska** LB 754 finally implemented (top 5.20%→4.55%, 4→3 brackets)
- **New York** Chapter 59/2025 Part A (bottom-5 rate cuts effective 1/1/26)
- **Oklahoma** HB 2764 (6→4 brackets, top 4.75%→4.50%, +0% zero-bracket)
- **Maine** new 2% millionaire surcharge added
- **Missouri** HB 798 (private pension exemption 100%) + TY 2026 thresholds

### TriSTAR #4 caught a real error
ChatGPT flagged MO bracket mismatch; Gemini missed it (matched the MO DOR
"year-changes" page which actually shows TY 2025). Direct PDF verification
confirmed ChatGPT correct. Fix committed `897d7ef`.
