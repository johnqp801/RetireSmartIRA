T4's `frontier.nonDominated` invariant fired on **3 profiles on this branch** (which already has the I3 fix): the heir-frontier plots strictly **dominated** points — more owner tax AND less to heirs as heir weight rises. Independently reproduced by a second reviewer; **material, not noise**:
- `mfj-b6-ca-verylarge-old`: +$14,295 tax / −$191,285 heirs
- `single-c3-nj-shorthorizon`: +$33,408 / −$33,328
- `mfj-c3-il-shorthorizon`: +$141,358 / −$59,683

**Root cause (corrected from the first LLM explanation — the exact failure mode the harness exists to prevent):** these profiles do NOT converge (hit iteration cap), so `keepBestOfCandidates` de-domination DOES run and STILL leaves a dominated frontier. Per-λ candidate scoring never enforces **cross-λ** Pareto non-domination. Fix = a final cross-λ Pareto-repair sweep, NOT removing the `!greedyConverged` gate. Deeper than I3. Saved to memory `frontier-cross-lambda-domination.md`; related to the known greedy/iteration-cap pathology ([[over-conversion-brake-ineffective]], INV13).
