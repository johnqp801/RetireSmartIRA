## Confirmed real work (build in 2.1)
- **NIIT is ABSENT from the multi-year engine** — `TaxCalculationEngine.calculateNIIT()` (~L736) exists but `ProjectionEngine` never calls it, so shipped multi-year federal totals OMIT NIIT. Pre-existing accuracy gap. Fix as Phase 0 (own `niit` channel, not folded into `federal`).
- **Taxable SS computed but not surfaced** on `YearRecommendation` (`ProjectionEngine` ~L539). Surface as a primitive.
- **Owner IRA vs 401(k) is LUMPED** in the multi-year engine: `MultiYearInputAdapter` (~L97) rolls owner traditional IRA + 401(k) into `primaryTrad`/`spouseTrad` via `.isTraditionalType`. The `AccountType` enum distinguishes them (AccountModels.swift:51), but the adapter collapses it. QCD needs IRA-only sourcing → split the owner pool into owner-IRA vs owner-401(k) per spouse. QCD-eligible = owner traditional IRA + (already-separate) inherited traditional IRA buckets; NEVER 401(k). RMD aggregation groups differ (own IRA / inherited IRA / 401(k) each separate).
- **Bracket-fill room is naive** in `cliffCandidates()` (`OptimizationEngine.swift:176`): `top − baselineTaxableIncome`. The existing optimizer is saved by full-stack candidate *ranking*, but a DETERMINISTIC fill-to-bracket ladder must **root-find** the conversion that lands *ordinary* taxable income at the ordinary-bracket top after SS/senior-phaseout/cap-gain nonlinearity (~$5–15k error otherwise).

## Tax-breakdown primitives (for the consequence decomposition)
`TaxBreakdown` (MultiYearValueTypes.swift:14) = `federal`, `state`, `irmaa`, `acaPremiumImpact`. `YearRecommendation` = `agi`, `taxableIncome`, `acaMagi`, `irmaaMagi`, `medicareEnrolledCount`. So state/IRMAA/ACA/MAGI/Medicare-count are already separable; NIIT + taxable-SS must be added. Path-level channel deltas (2.1.0) are cheap (subtract the two comparison paths); per-conversion attribution (2.1.1) is the finite-difference work.

## Phase 1a landed (2026-07-10) — owner IRA/401k split + engine env notes
- **Owner traditional is now split** into IRA vs 401(k) per spouse: `AccountSnapshot.primaryTraditionalIRA/primaryTraditional401k/spouseTraditionalIRA/spouseTraditional401k` (stored) with computed `.primaryTraditional`/`.spouseTraditional` (= IRA+401k); `TradBucket` value type (ira/k401/total/debit[401k-first]/grow/credit401k); `ProjectionEngine` tracks `var primary/spouse: TradBucket`. Behavior-preserving (full suite 1713 green). Non-QCD debits are 401(k)-first to preserve the IRA for QCDs.
- **1c QCD gotcha:** `MultiYearInputAdapter` folds metadata-LESS inherited-traditional IRAs into the owner **IRA** portion (to preserve the combined total). Inherited IRAs are a DISTINCT QCD aggregation group — when QCD sourcing lands in 1c, the QCD-eligible IRA balance must NOT silently include that folded inherited balance. (Metadata-carrying inherited IRAs are already separate `inheritedAccounts` buckets.)
- **Persistence:** `AccountSnapshot`/`YearRecommendation` are `Codable` but NOT serialized to disk/UserDefaults in production — the custom split-migration decode is latent/future-proof (test-exercised only).
- **Test env:** `xcodebuild test` STDOUT undercounts large Swift-Testing suites; use `-resultBundlePath` + `xcresulttool` for the authoritative count (Phase 1a full suite = 1713 cases).
