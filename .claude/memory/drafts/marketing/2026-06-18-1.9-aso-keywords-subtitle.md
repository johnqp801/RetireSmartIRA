### Changes from the prior field
Prior: `inherited,QCD,NIIT,ACA,subsidy,Medicare,surcharge,conversion,drawdown,MAGI,widow,survivor,senior,2026` (was **101 chars — 1 over the limit**, so "2026" was being truncated).
- **Removed `senior`** — generic, highly competitive, low conversion; weakest term.
- **Removed `widow`** — overlapped `survivor` (both target the survivor-bracket / widow's-penalty feature).
- **Added `pension`** — reflects the 1.9 NJ + NY pension-exclusion accuracy work; captures "pension tax" / "pension exclusion" searches.
- **Added `tax`** — was absent from name AND subtitle (a real gap for a tax app). As a broad connector it auto-combines with every other token into high-value phrases: "pension tax," "Medicare tax," "Roth conversion tax," "inherited tax," "2026 tax," etc. Only 3 chars; you rank for the combinations, not "tax" alone.

### Kept (on-brand, low-competition, high-intent for the detail-oriented-retiree niche)
inherited, QCD, NIIT, MAGI, surcharge, conversion, drawdown, ACA, subsidy, Medicare, survivor. These auto-combine with the name/subtitle (subtitle "Roth" + keyword "conversion" = "Roth conversion"; "Medicare" + "surcharge"; "ACA" + "subsidy").

## Subtitle decision — KEEP AS-IS for 1.9

Current subtitle: **`Roth, RMD & IRMAA Calculator`** (28 chars). **Decision: leave unchanged for 1.9.**

**Why:** the named trio (Roth/RMD/IRMAA) is specific, differentiated, and still accurately headlines the app; the drawdown feature belongs in release notes / description body / screenshots, not the 30 chars that drive discovery. "drawdown" is a weaker, less-defensible search term (Boldin/RetIQ own it).

**The one word to revisit later: `Calculator` vs `Planner`.** Real ASO tradeoff:
- Keep `Calculator` — captures high-intent searches ("RMD calculator," "IRMAA calculator") that convert well.
- Change to `Planner` — better reflects the app's depth + the new drawdown breadth (and the "name undersells breadth" point Ron raised).

**Plan:** treat `Roth, RMD & IRMAA Planner` as a deliberate **post-1.9 A/B test**, not a 1.9 change — watch "RMD/IRMAA calculator" impressions + downloads for ~2 weeks to isolate the ASO impact instead of confounding it with the feature launch.
