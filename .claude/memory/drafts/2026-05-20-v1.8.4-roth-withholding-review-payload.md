### Engine code

```swift
enum RothConversionWithholdingMode: String, Codable {
    case paidFromOutside        // default
    case withheldFromConversion
}

// On DataManager:
var scenarioTotalRothConversion: Double { yourRothConversion + (enableSpouse ? spouseRothConversion : 0) }
var rothConversionWithholdingMode: RothConversionWithholdingMode  // user-set
var rothConversionFederalWithholdingRate: Double  // user-set, default 0.24

var scenarioRothConversionWithholdingAmount: Double {
    switch rothConversionWithholdingMode {
    case .paidFromOutside:
        return 0
    case .withheldFromConversion:
        return scenarioTotalRothConversion * rothConversionFederalWithholdingRate
    }
}

var scenarioRothConversionNetAmount: Double {
    scenarioTotalRothConversion - scenarioRothConversionWithholdingAmount
}
```

In `applyRetirementExemptions` (state-tax engine), the PA / IL / MS conversion exemption logic is now:

```swift
switch state {
case .pennsylvania:
    // PA Ans 274 withholding caveat: exemption applies only if full balance
    // is deposited into the Roth. Withheld portion is PA-taxable.
    let netExempt = max(0, scenarioRothConversionAmount - scenarioRothConversionWithholdingAmount)
    adjusted -= netExempt
case .illinois, .mississippi:
    // No equivalent "full balance" rule documented in IL Pub 120 / MS Code
    // §27-7-15(4)(j). Full exemption preserved regardless of withholding.
    adjusted -= scenarioRothConversionAmount
default:
    break
}
```

Federal tax on the conversion is UNAFFECTED by withholding mode — the conversion is fully federally taxable either way; the withholding choice is a cash-flow question (where the tax dollars come from), not a tax-owed question.

The heir-comparison card adds a transparency note when withhold mode is active: "your heir actually inherits $NET in the Roth — the family-tax math is unchanged (you pay the same federal tax either way), but the Roth balance is smaller."

### Primary-source quotes

**PA DOR Answer ID 274:**
> A Roth conversion is not a taxable event in PA in the conversion year, provided the full pre-tax balance is deposited into the Roth. If any amount is withheld for federal tax, that withheld amount IS PA-taxable as a distribution.

**IL Publication 120:** treats trustee-to-trustee Roth conversions as not subject to IL income tax in the conversion year. No documented "full balance" requirement.

**MS Code § 27-7-15(4)(j):** retirement distributions are exempt from MS income tax. Roth conversion treated as a distribution event, exempt from MS tax. No documented "full balance" requirement.

**Federal tax liability:** A Roth conversion is fully includible in gross income in the conversion year (IRC § 408A(d)(3)(A)). Whether the user pays the resulting federal tax from outside money or via withholding from the conversion itself does not change the federal tax liability — only the cash-flow source.

**Early withdrawal penalty (under 59½):** If a taxpayer is under 59½ and federal tax is withheld from a Roth conversion, the IRS may treat the withheld amount as an early distribution subject to the 10% additional tax under IRC § 72(t). The app does NOT model this penalty; it surfaces a warning banner.
