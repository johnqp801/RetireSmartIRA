# Multi-Year Tax Strategy Engine — Plan A (Engine Only)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the pure-calculation `MultiYearTaxStrategyEngine` that produces a 30-year retirement-tax recommendation, with stress-test bands, widow-stress callout, and SS-claim-age nudge — fully testable in isolation against published reference scenarios.

**Architecture:** Three internal layers — `ProjectionEngine` (year-forward simulation), `OptimizationEngine` (the optimizer; architecture chosen by Phase 0 perf prototype), and stress-test wrappers (`StressTestRunner`, `WidowStressTest`, `SSClaimNudge`). Top-level `MultiYearTaxStrategyEngine` coordinates them. All types are pure value types with no SwiftUI dependencies.

**Tech Stack:** Swift 6, Xcode 16 with `PBXFileSystemSynchronizedRootGroup` (filesystem-driven file detection — new files appear in target automatically), XCTest for unit tests, Decimal for monetary precision.

**Spec:** [docs/superpowers/specs/2026-05-02-2.0-multi-year-tax-strategy-design.md](../specs/2026-05-02-2.0-multi-year-tax-strategy-design.md) (commit `ec9c9d0`).

**Successor plan (Plan B — UI):** to be written after engine ships.

## Context for the implementer

You are implementing the engine layer of RetireSmartIRA 2.0. The engine is a **pure-calculation library** — no SwiftUI, no `DataManager`, no async, no I/O. It takes a `Scenario` (existing 1.9 type, lives in `TaxModels.swift`) plus a new `MultiYearAssumptions` struct, and returns a `MultiYearStrategyResult` with a recommended 30-year path.

The codebase is **flat** — all Swift files live directly under `RetireSmartIRA/` (no nested `Engines/` or `Models/` folders). New files added to the filesystem are picked up automatically by Xcode's file-system-synchronized group. Place all new files at the top level of `RetireSmartIRA/`.

Tests live in `RetireSmartIRATests/`, also flat.

**Existing engines to model after:** `TaxCalculationEngine.swift`, `ACASubsidyEngine.swift`, `MedicareCostEngine.swift`, `SSCalculationEngine.swift`. All are pure-calc structs/enums with static methods. **Read these before writing tests** — they establish the conventions you should follow (typed AGI variants, Decimal money values, explicit-parameter style).

**Existing types you'll consume:**
- `Scenario` — defined in `TaxModels.swift`. The static-input container for a planning scenario.
- `FederalAGI`, `ACAMAGI`, `IRMAAMAGI` — strongly-typed AGI variants. Defined in tax models. Use these instead of raw `Decimal`/`Double` for AGI quantities to prevent unit confusion.
- `FilingStatus` enum.
- `AccountType` enum (in `AccountModels.swift`) with cases `traditionalIRA`, `rothIRA`, `taxable`, `hsa` (verify exact case names in Task 0.2).

**Money convention:** verify in Task 0.2 — likely `Decimal` throughout. If a `Money` typealias exists, use it; otherwise use `Decimal` literally.

**Branch strategy:** Phase 0 work happens on the `1.9/features-bundle` branch (where PR #4 lives) for Task 0.1 only. Tasks 0.2 onward happen on a new `2.0/multi-year-engine` branch in a new worktree.

import Foundation

// Discretization
let yearCount = 30
let bracketBucketCount = 7      // 10/12/22/24/32/35/37
let irmaaTierCount = 6
let rothBalanceBucketCount = 20

// State: (year, bracketBucket, irmaaTier, rothBalanceBucket) -> Decimal (best lifetime tax from this state)
let totalStates = yearCount * bracketBucketCount * irmaaTierCount * rothBalanceBucketCount
print("Total states: \(totalStates)")

// Allocate DP table
var dp = [Decimal](repeating: 0, count: totalStates)

// Action set: Roth conversion amount discretized to 10 levels per state
let actionLevels = 10

// Time the backward induction
let start = Date()

for year in (0..<yearCount).reversed() {
    for bracket in 0..<bracketBucketCount {
        for irmaa in 0..<irmaaTierCount {
            for roth in 0..<rothBalanceBucketCount {
                let stateIndex = ((year * bracketBucketCount + bracket) * irmaaTierCount + irmaa) * rothBalanceBucketCount + roth

                var bestCost = Decimal.greatestFiniteMagnitude
                for action in 0..<actionLevels {
                    // Stub transition: synthetic cost based on action + lookup of next state
                    let transitionCost = Decimal(action * 1000)  // placeholder
                    let nextStateIndex: Int
                    if year == yearCount - 1 {
                        nextStateIndex = stateIndex  // terminal
                        bestCost = min(bestCost, transitionCost)
                    } else {
                        // Simple next-state mapping (synthetic)
                        nextStateIndex = (((year + 1) * bracketBucketCount + bracket) * irmaaTierCount + irmaa) * rothBalanceBucketCount + roth
                        bestCost = min(bestCost, transitionCost + dp[nextStateIndex])
                    }
                }
                dp[stateIndex] = bestCost
            }
        }
    }
}

let elapsed = Date().timeIntervalSince(start)
print("DP backward induction: \(elapsed) seconds")
print("DP[0] = \(dp[0])")

import Foundation

// MARK: - Withdrawal ordering rule (user-facing preset)

public enum WithdrawalOrderingRule: String, Codable, CaseIterable, Equatable {
    case taxEfficient = "tax_efficient"
    case depleteTradFirst = "deplete_trad_first"
    case preserveRoth = "preserve_roth"
    case proportional = "proportional"

    public static let `default`: WithdrawalOrderingRule = .taxEfficient
}

// MARK: - Spouse identity

public enum SpouseID: String, Codable, CaseIterable, Equatable {
    case primary
    case spouse
}

// MARK: - Lever actions (per-year recommended moves)

public enum LeverAction: Codable, Equatable {
    case rothConversion(amount: Decimal)
    case traditionalWithdrawal(amount: Decimal)
    case taxableWithdrawal(amount: Decimal)
    case rothWithdrawal(amount: Decimal)
    case hsaContribution(amount: Decimal)

//
//  ProjectionEngine.swift
//  RetireSmartIRA
//
//  Year-by-year forward simulation given a sequence of LeverActions.
//  Pure-calculation; delegates tax / Medicare / ACA / SS math to existing 1.9 engines.
//

import Foundation

public struct ProjectionEngine {

    public init() {}

    /// Project the scenario forward for each year in `actionsPerYear` (sorted ascending).
    /// Returns a `YearRecommendation` per year.
    ///
    /// Year-end semantics:
    ///   1. Apply year's actions (Roth conversions, withdrawals, contributions) to balances.
    ///   2. Apply auto-funded expense withdrawals using the withdrawalOrderingRule.
    ///   3. Apply growth (investmentGrowthRate) to remaining balances.
    ///   4. Compute AGI, taxes, etc. on the year's events.
    public func project(
        scenario: Scenario,
        assumptions: MultiYearAssumptions,
        actionsPerYear: [Int: [LeverAction]]
    ) -> [YearRecommendation] {
        var balances = startingBalances(from: scenario)
        var currentAge = startingAge(from: scenario)
        var results: [YearRecommendation] = []

        let sortedYears = actionsPerYear.keys.sorted()
        for year in sortedYears {
            let actions = actionsPerYear[year] ?? []

            // 1. Apply explicit actions
            balances = applyActions(actions, to: balances)

            // 2. Auto-fund expenses if needed (TODO: hook into withdrawalOrderingRule)
            let expense = assumptions.perYearExpenseOverrides[year] ?? defaultExpense(for: year, scenario: scenario, assumptions: assumptions)
            balances = fundExpenses(amount: expense, from: balances, rule: assumptions.withdrawalOrderingRule)

            // 3. Grow remaining balances
            balances = applyGrowth(balances, rate: Decimal(assumptions.investmentGrowthRate))

            // 4. Compute year's tax breakdown
            let taxYear = computeYearTax(
                year: year,
                actions: actions,
                expenses: expense,
                scenario: scenario,
                assumptions: assumptions,
                age: currentAge
            )

            results.append(YearRecommendation(
                year: year,
                agi: taxYear.agi,
                acaMagi: taxYear.acaMagi,
                irmaaMagi: taxYear.irmaaMagi,
                taxableIncome: taxYear.taxableIncome,
                taxBreakdown: taxYear.taxBreakdown,
                endOfYearBalances: balances,
                actions: actions
            ))

            currentAge += 1
        }

        return results
    }

    // MARK: - Internal helpers

    /// Build initial AccountSnapshot from the Scenario's account state.
    /// Adapts to the actual Scenario shape — verify field names match (e.g., .traditionalIRABalance vs .traditionalIRA).
    private func startingBalances(from scenario: Scenario) -> AccountSnapshot {
        // PLACEHOLDER mapping — confirm exact Scenario property names from TaxModels.swift
        return AccountSnapshot(
            traditionalIRA: scenarioTraditionalIRA(scenario),
            rothIRA: scenarioRothIRA(scenario),
            taxable: scenarioTaxable(scenario),
            hsa: scenarioHSA(scenario)
        )
    }

    private func startingAge(from scenario: Scenario) -> Int {
        // Adapt to actual property name (e.g., scenario.currentAge or computed from birth year)
        return scenarioCurrentAge(scenario)
    }

    private func applyActions(_ actions: [LeverAction], to balances: AccountSnapshot) -> AccountSnapshot {
        var b = balances
        for action in actions {
            switch action {
            case .rothConversion(let amount):
                b = AccountSnapshot(
                    traditionalIRA: b.traditionalIRA - amount,
                    rothIRA: b.rothIRA + amount,
                    taxable: b.taxable,
                    hsa: b.hsa
                )
            case .traditionalWithdrawal(let amount):
                b = AccountSnapshot(
                    traditionalIRA: b.traditionalIRA - amount,
                    rothIRA: b.rothIRA,
                    taxable: b.taxable,
                    hsa: b.hsa
                )
            case .taxableWithdrawal(let amount):
                b = AccountSnapshot(
                    traditionalIRA: b.traditionalIRA,
                    rothIRA: b.rothIRA,
                    taxable: b.taxable - amount,
                    hsa: b.hsa
                )
            case .rothWithdrawal(let amount):
                b = AccountSnapshot(
                    traditionalIRA: b.traditionalIRA,
                    rothIRA: b.rothIRA - amount,
                    taxable: b.taxable,
                    hsa: b.hsa
                )
            case .hsaContribution(let amount):
                b = AccountSnapshot(
                    traditionalIRA: b.traditionalIRA,
                    rothIRA: b.rothIRA,
                    taxable: b.taxable - amount,
                    hsa: b.hsa + amount
                )
            case .fourOhOneKContribution(let amount):
                b = AccountSnapshot(
                    traditionalIRA: b.traditionalIRA + amount,
                    rothIRA: b.rothIRA,
                    taxable: b.taxable - amount,
                    hsa: b.hsa
                )
            case .deferSocialSecurity, .claimSocialSecurity:
                break  // Affects income calc, not balances directly
            }
        }
        return b
    }

    private func fundExpenses(amount: Decimal, from balances: AccountSnapshot, rule: WithdrawalOrderingRule) -> AccountSnapshot {
        guard amount > 0 else { return balances }
        var b = balances
        var remaining = amount

        let order: [WritableKeyPath<AccountSnapshot, Decimal>]  // declared abstractly
        // AccountSnapshot has let properties, so we can't use writable keypaths directly.
        // Instead use a tuple or rebuild snapshots. For clarity, rebuild step-by-step:
        switch rule {
        case .taxEfficient:
            // Simplified rule: taxable first, then traditional (let bracket-fill be optimizer's job),
            // then Roth as last resort. Real bracket-fill happens in OptimizationEngine.
            (b, remaining) = drainAccount(.taxable, b, remaining)
            (b, remaining) = drainAccount(.traditionalIRA, b, remaining)
            (b, remaining) = drainAccount(.rothIRA, b, remaining)
        case .depleteTradFirst:
            (b, remaining) = drainAccount(.traditionalIRA, b, remaining)
            (b, remaining) = drainAccount(.taxable, b, remaining)
            (b, remaining) = drainAccount(.rothIRA, b, remaining)
        case .preserveRoth:
            (b, remaining) = drainAccount(.taxable, b, remaining)
            (b, remaining) = drainAccount(.traditionalIRA, b, remaining)
            (b, remaining) = drainAccount(.rothIRA, b, remaining)  // last resort
        case .proportional:
            // Take 1/3 from each non-Roth bucket; Roth last.
            let third = amount / 3
            (b, _) = drainAccount(.traditionalIRA, b, min(b.traditionalIRA, third))
            (b, _) = drainAccount(.taxable, b, min(b.taxable, third))
            (b, _) = drainAccount(.hsa, b, min(b.hsa, third))
            // Re-compute remaining based on actual drains
            let drainedTotal = balances.total - b.total
            remaining = max(0, amount - drainedTotal)
            (b, remaining) = drainAccount(.rothIRA, b, remaining)
        }
        return b
    }

    private enum AccountKind { case traditionalIRA, rothIRA, taxable, hsa }

    private func drainAccount(_ kind: AccountKind, _ balances: AccountSnapshot, _ amount: Decimal) -> (AccountSnapshot, Decimal) {
        guard amount > 0 else { return (balances, 0) }
        switch kind {
        case .traditionalIRA:
            let drawn = min(balances.traditionalIRA, amount)
            return (AccountSnapshot(
                traditionalIRA: balances.traditionalIRA - drawn,
                rothIRA: balances.rothIRA, taxable: balances.taxable, hsa: balances.hsa
            ), amount - drawn)
        case .rothIRA:
            let drawn = min(balances.rothIRA, amount)
            return (AccountSnapshot(
                traditionalIRA: balances.traditionalIRA,
                rothIRA: balances.rothIRA - drawn, taxable: balances.taxable, hsa: balances.hsa
            ), amount - drawn)
        case .taxable:
            let drawn = min(balances.taxable, amount)
            return (AccountSnapshot(
                traditionalIRA: balances.traditionalIRA,
                rothIRA: balances.rothIRA, taxable: balances.taxable - drawn, hsa: balances.hsa
            ), amount - drawn)
        case .hsa:
            let drawn = min(balances.hsa, amount)
            return (AccountSnapshot(
                traditionalIRA: balances.traditionalIRA,
                rothIRA: balances.rothIRA, taxable: balances.taxable, hsa: balances.hsa - drawn
            ), amount - drawn)
        }
    }

    private func applyGrowth(_ balances: AccountSnapshot, rate: Decimal) -> AccountSnapshot {
        let factor = 1 + rate
        return AccountSnapshot(
            traditionalIRA: balances.traditionalIRA * factor,
            rothIRA: balances.rothIRA * factor,
            taxable: balances.taxable * factor,
            hsa: balances.hsa * factor
        )
    }

    private struct YearTaxResult {
        let agi: Decimal
        let acaMagi: Decimal?
        let irmaaMagi: Decimal?
        let taxableIncome: Decimal
        let taxBreakdown: TaxBreakdown
    }

    private func computeYearTax(
        year: Int,
        actions: [LeverAction],
        expenses: Decimal,
        scenario: Scenario,
        assumptions: MultiYearAssumptions,
        age: Int
    ) -> YearTaxResult {
        // Decompose actions into income components:
        let rothConversionIncome = actions.compactMap { if case .rothConversion(let a) = $0 { return a } else { return nil } }.reduce(0, +)
        let tradWithdrawalIncome = actions.compactMap { if case .traditionalWithdrawal(let a) = $0 { return a } else { return nil } }.reduce(0, +)
        // Plus auto-funded trad withdrawals from fundExpenses — must be tracked separately;
        // simplification for v0: approximate by re-computing what fundExpenses pulled.
        // (Refine in iteration; reference scenarios will catch this.)

//
//  ConstraintAcceptor.swift
//  RetireSmartIRA
//
//  Detects soft-constraint hits (IRMAA tiers, ACA cliff, bracket overruns) along a candidate
//  multi-year path. Used by OptimizationEngine to decide whether to accept a path's trade-offs.
//

import Foundation

public struct ConstraintAcceptor {

    public init() {}

    // MARK: IRMAA

    public func detectIRMAAHits(
        yearMAGI: [(year: Int, magi: Decimal)],
        filingStatus: FilingStatus
    ) -> [ConstraintHit] {
        var hits: [ConstraintHit] = []
        for (year, magi) in yearMAGI {
            // Tier thresholds applicable to filing year + 2 (lookback) — but for v2.0 we
            // approximate by matching tier thresholds to the year of MAGI. Refine in 2.1
            // if reference scenarios show meaningful drift.
            let tier = MedicareCostEngine.irmaaTier(magi: magi, filingStatus: filingStatus)
            if tier > 0 {
                let cost = MedicareCostEngine.totalIRMAACost(magi: magi, filingStatus: filingStatus)
                hits.append(ConstraintHit(
                    year: year,
                    type: .irmaaTier(level: tier),
                    cost: cost,
                    acceptanceRationale: ""  // set by caller after lifetime-savings comparison
                ))
            }
        }
        return hits
    }

    // MARK: ACA

    public func detectACAHits(
        yearMAGI: [(year: Int, magi: Decimal, age: Int)],
        householdSize: Int
    ) -> [ConstraintHit] {
        var hits: [ConstraintHit] = []
        for (year, magi, age) in yearMAGI {
            guard age < 65 else { continue }
            let fpl = ACASubsidyEngine.federalPovertyLevel(year: year, householdSize: householdSize)
            let threshold = fpl * 4
            if magi > threshold {
                // Cliff trip: full subsidy lost. Cost estimated as the prior year's subsidy.
                let cliffCost = ACASubsidyEngine.estimatedSubsidy(
                    magi: threshold,  // estimate at the threshold
                    householdSize: householdSize
                )
                hits.append(ConstraintHit(
                    year: year,
                    type: .acaCliff,
                    cost: cliffCost,
                    acceptanceRationale: ""
                ))
            }
        }
        return hits
    }

    // MARK: Bracket overruns

    public func detectBracketOverruns(
        yearTaxableIncome: [(year: Int, taxableIncome: Decimal)],
        filingStatus: FilingStatus
    ) -> [ConstraintHit] {
        var hits: [ConstraintHit] = []
        for (year, taxable) in yearTaxableIncome {
            let brackets = TaxCalculationEngine.federalBrackets(year: year, filingStatus: filingStatus)
            // Find the marginal bracket and detect if income spans a 12->22 or 22->24 jump
            // (the painful jumps; ignore 32->35 which is small).
            for (i, bracket) in brackets.enumerated() {
                let nextThreshold = i + 1 < brackets.count ? brackets[i + 1].lowerBound : Decimal.greatestFiniteMagnitude
                let isPainfulJump = (bracket.rate == 12 && i + 1 < brackets.count && brackets[i + 1].rate == 22) ||
                                    (bracket.rate == 22 && i + 1 < brackets.count && brackets[i + 1].rate == 24)
                if isPainfulJump && taxable > nextThreshold {
                    let overrunAmount = taxable - nextThreshold
                    let costDelta = overrunAmount * Decimal((brackets[i + 1].rate - bracket.rate) / 100)
                    hits.append(ConstraintHit(
                        year: year,
                        type: .bracketOverrun(fromBracket: bracket.rate, toBracket: brackets[i + 1].rate),
                        cost: costDelta,
                        acceptanceRationale: ""
                    ))
                    break  // only flag the most painful overrun per year
                }
            }
        }
        return hits
    }

    // MARK: Rationale formatting

    public func formatAcceptanceRationale(lifetimeSavings: Decimal, constraintCost: Decimal) -> String {
        let formatter = NumberFormatter()
        formatter.numberStyle = .currency
        formatter.currencyCode = "USD"
        formatter.maximumFractionDigits = 0
        let savingsStr = formatter.string(from: lifetimeSavings as NSDecimalNumber) ?? "$\(lifetimeSavings)"
        let costStr = formatter.string(from: constraintCost as NSDecimalNumber) ?? "$\(constraintCost)"
        return "Lifetime savings \(savingsStr) > constraint cost \(costStr)"
    }
}

**Algorithm by scope (high-level):**

- **Scope E (DP):** state = `(year, IRMAA-tier, bracket-bucket, Roth-balance-bucket)`. Backward induction. Action = Roth conversion amount discretized. Cost-to-go = lifetime tax. Reconstruct path forward.
- **Scope C+D (lookahead-greedy):** for each year, try a small set of candidate Roth conversion amounts (e.g., $0, $20K, $40K, fill-12%-bracket, fill-22%-bracket). - **Scope B (pure greedy):** for each year, maximize current-year tax efficiency under constraint awareness (don't trip an IRMAA tier unless lifetime savings > cost). No multi-year lookahead.

//
//  MultiYearTaxStrategyEngine.swift
//  RetireSmartIRA
//
//  Top-level entry point for multi-year tax strategy. Coordinates the optimization,
//  stress tests, widow scenario, and SS-nudge sub-engines.
//

import Foundation

public struct MultiYearTaxStrategyEngine {

    public init() {}

    public func compute(
        scenario: Scenario,
        assumptions: MultiYearAssumptions
    ) -> MultiYearStrategyResult {

        // 1. Main optimization
        let optEngine = OptimizationEngine()
        let optResult = optEngine.optimize(scenario: scenario, assumptions: assumptions)

        // 2. Sensitivity bands (only if enabled)
        let bands: SensitivityBands
        if assumptions.stressTestEnabled {
            bands = StressTestRunner().run(scenario: scenario, assumptions: assumptions)
        } else {
            bands = SensitivityBands(
                optimistic: optResult.recommendedPath,
                average: optResult.recommendedPath,
                pessimistic: optResult.recommendedPath
            )
        }

        // 3. Widow stress test (always runs; separate concern per spec)
        let widowImpact = WidowStressTest().run(scenario: scenario, assumptions: assumptions)

        // 4. SS claim-age nudge
        let ssFlag = SSClaimNudge().compute(scenario: scenario, assumptions: assumptions)

        return MultiYearStrategyResult(
            recommendedPath: optResult.recommendedPath,
            tradeOffsAccepted: optResult.tradeOffsAccepted,
            sensitivityBands: bands,
            widowStressDelta: widowImpact,
            ssClaimNudge: ssFlag
        )
    }
}
