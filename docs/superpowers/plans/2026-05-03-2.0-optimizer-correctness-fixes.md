# 2.0 Optimizer Correctness Fixes — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix three correctness bugs in `OptimizationEngine.swift` (Terminal Tax Illusion, Greedy Trap, Bracket-Edge Blindness) so the V2.0 multi-year optimizer actually discovers Roth conversion ladders instead of recommending "do nothing" in nearly all scenarios.

**Architecture:** Add two fields to `MultiYearAssumptions`. Wrap the existing forward-greedy inner loop in a fixed-point iteration outer loop with convergence detection. Compute per-year baseline MAGI inside the inner loop (Gemini correction). Generate cliff-aware candidates (IRMAA / ACA / bracket boundaries) from `TaxYearConfig.config` with a $5K buffer for true cliffs. Add terminal liquidation tax to the objective function so end-of-horizon trad balance is no longer treated as free.

**Tech Stack:** Swift 5+, Swift Testing framework (`import Testing`), `XCTest`-free, Foundation. Spec at [docs/superpowers/specs/2026-05-03-2.0-optimizer-correctness-fixes-design.md](../specs/2026-05-03-2.0-optimizer-correctness-fixes-design.md).

## Spec deviations (read before implementing)

1. **No `MultiYearTaxThresholds.swift`.** Spec proposed a new constants file. Implementation reads directly from `TaxYearConfig.config` to avoid duplication and to guarantee the cliff candidate generator targets exactly the thresholds the engine actually penalizes.

2. **No CPI-projection of cliff thresholds.** Spec described `(1+cpi)^(year-baseYear)` projection. Implementation does NOT project, because the underlying engine (`TaxCalculationEngine.calculateIRMAA(magi:filingStatus:)`) uses unadjusted 2026 values for all projected years. Projecting cliff candidates ahead would aim at moving targets the engine doesn't actually penalize. The engine's lack of year-adjustment is a separate pre-existing concern; flagging as v2.1 work, out of scope here.

3. **`cliffCandidates` signature uses optional MAGI inputs.** `YearRecommendation.acaMagi` and `YearRecommendation.irmaaMagi` are `Double?` (nil when not relevant). Generator accepts optionals; nil means "skip generating candidates of that cliff type."

```swift
// MARK: - Cliff candidate generator
//
// Generates per-year Roth conversion amounts that land MAGI / taxable income
// at strategically useful targets:
//   - IRMAA Tier boundaries (with `cliffBuffer` margin below each)
//   - ACA 400% FPL (with `cliffBuffer` margin below)
//   - Ordinary tax bracket tops (NO buffer — these aren't cliffs, just fill targets)
//
// Returns the conversion-AMOUNT deltas, not absolute MAGI targets. Caller
// unions these with the static candidate set in optimize().
//
// Reads thresholds DIRECTLY from TaxYearConfig.config. Does NOT CPI-project,
// because the underlying engine (TaxCalculationEngine.calculateIRMAA) does
// not year-adjust either; we must aim at exactly what the engine penalizes.
//
// Filtering:
//   - Drops non-positive deltas (cliff already passed)
//   - Drops deltas > $500_000 (unreasonable single-year conversion size)
//
// `internal` (not `private`) so unit tests in CliffCandidateGeneratorTests
// can call it directly.
static func cliffCandidates(
    forYear year: Int,
    baselineIRMAAMagi: Double?,
    baselineACAMagi: Double?,
    baselineTaxableIncome: Double,
    filingStatus: FilingStatus,
    householdSize: Int,
    assumptions: MultiYearAssumptions
) -> [Double] {
    var candidates: [Double] = []
    let buffer = assumptions.cliffBuffer
    let cap: Double = 500_000

    // ─── IRMAA tier candidates (only if Medicare-relevant) ───
    if let irmaaMagi = baselineIRMAAMagi {
        // TaxYearConfig.config.irmaaTiers has 6 entries: tier 0 (no surcharge) + tiers 1-5.
        // We want fill-to-(threshold - buffer) for tiers 1-5 only (skip tier 0, which has
        // threshold == 0 and is the standard zone).
        for tierEntry in TaxYearConfig.config.irmaaTiers where tierEntry.tier > 0 {
            let threshold = filingStatus == .single
                ? tierEntry.singleThreshold
                : tierEntry.mfjThreshold
            let target = threshold - buffer
            let delta = target - irmaaMagi
            if delta > 0 && delta <= cap {
                candidates.append(delta)
            }
        }
    }

    // ─── ACA 400% FPL candidate (only if ACA-relevant) ───
    if let acaMagi = baselineACAMagi {
        // householdSizeToFPL is keyed by string ("1", "2", ...). Cap at 8 (config max).
        let key = String(min(householdSize, 8))
        if let fpl = TaxYearConfig.config.acaSubsidy2026.fpl2026.householdSizeToFPL[key] {
            let cliff = fpl * 4.0
            let target = cliff - buffer
            let delta = target - acaMagi
            if delta > 0 && delta <= cap {
                candidates.append(delta)
            }
        }
    }

    // ─── Ordinary tax bracket tops (no buffer) ───
    let brackets = TaxYearConfig.config.toTaxBrackets()
    let bracketArray = filingStatus == .single ? brackets.federalSingle : brackets.federalMarried
    // Bracket "top" for bracket i = bracket i+1's threshold. Last bracket has no top.
    for i in 0..<(bracketArray.count - 1) {
        let top = bracketArray[i + 1].threshold
        let delta = top - baselineTaxableIncome
        if delta > 0 && delta <= cap {
            candidates.append(delta)
        }
    }

    return candidates
}
```

```swift
// MARK: - Terminal liquidation tax helper
//
// Estimates the future tax burden of leftover Traditional balance at the
// end of the horizon. Critical for the optimizer's objective function:
// without this, the engine treats deferred trad as free wealth and biases
// toward "do nothing" (Bug 1 — Terminal Tax Illusion).
//
// Uses MultiYearAssumptions.terminalLiquidationTaxRate (default 0.22).
// For pre-Plan-B v2.0 ship, the rate is hardcoded in MultiYearAssumptions
// defaults; Plan B will surface it in onboarding.
private func terminalLiquidationTax(
    _ path: [YearRecommendation],
    rate: Double
) -> Double {
    guard let last = path.last else { return 0 }
    let trad = last.endOfYearBalances.primaryTraditional
             + last.endOfYearBalances.spouseTraditional
    return trad * rate
}
```
