# 2.0.1 — Senior Bonus Visibility + Chart Commentary Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface the OBBBA Senior Bonus card on the Tax Summary tab, and add a tap-to-reveal plain-language explanation popover to each of the five Multi-Year charts.

**Architecture:** Presentation-only. A pure `ChartCommentary` value type carries a title + body; each of the five chart models gains a deterministic `commentary` computed property derived from its own numbers (same philosophy as `PlanSummary.headline`). A single reusable `ChartInfoButton` renders the ⓘ affordance and its popover, wired into each chart view's header. The Senior Bonus card is a drop-in `SeniorBonusDeductionCard()` added to `DashboardView`.

**Tech Stack:** Swift 5 / SwiftUI, Swift Charts, XCTest. Xcode 16 project with `PBXFileSystemSynchronizedRootGroup` — new `.swift` files under `RetireSmartIRA/` and `RetireSmartIRATests/` are auto-included in their targets; NO `project.pbxproj` edits are required for new files.

## Global Constraints

- No changes to the tax/retirement engine. Presentation only. (spec: non-goals)
- Commentary is generated deterministically from the chart model. NO LLM, offline, instant. (spec: Part 2)
- No em dash character anywhere in user-facing copy. (project rule)
- Voice: plain, non-alarmist, matches existing on-tab copy. (spec: Part 2)
- Auto-generated financial text that is subtly wrong is worse than none: never claim IRMAA when there is none, never claim conversions when the path has none. Each `commentary` gets unit tests. (spec: Part 2 testing)
- Platform target: native macOS (not Catalyst) + iOS/iPadOS, iOS 18 / macOS 15. (project)
- A change is not done until the full suite is green (1,164+ tests). (project rule)
- Version bump to 2.0.1 / build 59 and release notes are handled as a SEPARATE release step, NOT in this plan.

**Test commands** (pure-model tests run on macOS; substitute an iOS Simulator destination if the local macOS test action is not configured):
- Single class: `xcodebuild test -scheme RetireSmartIRA -destination 'platform=macOS' -only-testing:RetireSmartIRATests/<Class> 2>&1 | tail -30`

```swift
import Foundation

// Deterministic, per-chart plain-language commentary. Each property reads only fields the model
// already holds and degrades gracefully on empty/degenerate input. Keep the voice plain and
// non-alarmist; no em dashes. Auto-generated financial text that is subtly wrong is worse than
// none, so every branch here is covered by ChartCommentaryTests.

extension BalancesChart {
    var commentary: ChartCommentary {
        let base = "Each line is a projected balance under this plan: your Traditional IRA or 401(k), your Roth, and your taxable accounts, shown year by year."
        let trend: String
        if let last = points.last, last.roth > last.traditional, last.roth > 0 {
            trend = " By the end of the plan, more of your money sits in the tax-free Roth than in the Traditional account, which is still taxed when withdrawn."
        } else if let last = points.last, last.traditional > 0 {
            trend = " The Traditional balance is the part that still faces income tax when it is withdrawn or passed to heirs."
        } else {
            trend = ""
        }
        let band = hasBand
            ? " The shaded band shows how the total shifts under higher or lower constant growth. It is growth-assumption sensitivity, not a probability or the odds of success."
            : ""
        return ChartCommentary(title: "Account balances over time", body: base + trend + band)
    }
}
```

Create `RetireSmartIRA/ChartInfoButton.swift`:

```swift
import SwiftUI

/// A quiet ⓘ affordance that presents a chart's plain-language explanation in a popover.
/// Reused by every Multi-Year chart header. Presentational only; the text comes from the
/// chart model's deterministic `commentary` (see ChartCommentary+Models).
struct ChartInfoButton: View {
    let commentary: ChartCommentary
    @State private var showing = false

    var body: some View {
        Button {
            showing = true
        } label: {
            Image(systemName: "info.circle")
                .font(.footnote)
                .foregroundStyle(.secondary)
        }
        .buttonStyle(.borderless)
        .accessibilityLabel("Explain this chart")
        .accessibilityHint(commentary.body)
        .popover(isPresented: $showing) {
            VStack(alignment: .leading, spacing: 8) {
                Text(commentary.title).font(.headline)
                Text(commentary.body).font(.callout).foregroundStyle(.secondary)
            }
            .padding()
            .frame(maxWidth: 320, alignment: .leading)
            .presentationCompactAdaptation(.popover)
        }
    }
}
```
