## Pre-Execution Design Decisions (added 2026-05-17 after plan-review)

The plan below was authored against the spec but surfaced four design questions during planning. John resolved them after review. **Apply these deltas as you execute the relevant tasks — they override the corresponding sections below.**

### Decision A — L3 spouse-heir gating (applies to Task 3)
**Gate the heir-bracket comparison card so it only renders when the heir is NOT a spouse.** Spouse heirs can roll the inherited IRA into their own and use their own RMD timeline; the 10-year drain assumption baked into the comparison is wrong for them. Use `dataManager.legacyHeirTypeDescription` (or equivalent canonical accessor) to detect spouse and return `EmptyView()` in that case. Add a test asserting the card is hidden when heir type is spouse.

### Decision B — L3 hybrid "per $X" comparison display (applies to Task 3)
**Use a hybrid display for the conversion amount in the comparison:**
- When `scenarioTotalRothConversion >= $10,000`: show the live amount ("Converting $X this year saves your heirs ~$Y")
- When `scenarioTotalRothConversion < $10,000`: show illustrative constant ("Per $100K converted, your heirs save ~$Z")
Add a test for both branches at the $10K boundary.

### Decision C — L4 widow lifetime tax delta math (applies to Task 4)
**Ship the coarse 0.85 single-filer approximation as planned, BUT add an inline tooltip on the sensitivity slider readout:** *"Approximate single-filer adjustment. V2.0 will model this year-by-year through the multi-year engine."* This is honest about the approximation, sets up V2.0 anticipation, and avoids over-engineering for 1.8.2. Engine year-by-year re-run is explicitly deferred to V2.0.

### Decision D — L2 taxable-brokerage-balance toggle (applies to Task 2; adds ~0.25 day to phase)
**Add a `hasTaxableBrokerage: Bool` toggle to the user profile (default `false`), surfaced in Profile/Settings, and gate the L2 0% LTCG card on it.** This:
1. Adds a clean signal that unlocks future surfaces (state-tax modeling, NIIT precision, etc.)
2. Removes the "card always visible" awkwardness for non-brokerage users
3. Costs ~0.25 day (one bool, one toggle UI, one accessor, one test)

**Plan delta:** Insert a sub-task "2.0" before the current Task 2 steps that adds the toggle to `ProfileManager` (or equivalent persistence layer), wires a SettingsView row for it, and writes a round-trip persistence test. Then update Task 2's card-mounting condition from "unconditional" to `if dataManager.profile.hasTaxableBrokerage`.

### Effort impact

Phase 2 estimate was 7.75 days. Decisions A and B add minimal time (gate logic + one extra test each). Decision C adds zero time (tooltip is a one-liner). Decision D adds ~0.25 day. Revised Phase 2 estimate: **~8 days**.

```swift
/// Top of the 0% long-term capital gains bracket for the given filing status, using the current TaxYearConfig.
/// In 2026: $98,900 MFJ, $49,450 single.
static func ltcg0PercentTop(filingStatus: FilingStatus) -> Double {
    let brackets = default2026Brackets
    let capGains = filingStatus == .single ? brackets.federalCapGainsSingle : brackets.federalCapGainsMarried
    // Top of the 0% band = threshold where the 15% rate begins.
    if let firstNonZero = capGains.first(where: { $0.rate > 0 }) {
        return firstNonZero.threshold
    }
    return 0
}

/// Remaining headroom inside the 0% LTCG bracket, given current taxable income.
static func ltcg0PercentHeadroom(taxableIncome: Double, filingStatus: FilingStatus) -> Double {
    let top = ltcg0PercentTop(filingStatus: filingStatus)
    return max(0, top - taxableIncome)
}
```

```swift
    let heirMarginalRate: Double

/// Compare paying tax at the user's current marginal rate vs an heir paying tax at their
/// marginal rate after a SECURE-Act 10-year drain. Positive `netFamilyBenefit` means
/// converting now is cheaper for the family.
static func convertNowVsHeirComparison(
    conversionAmount: Double,
    userMarginalRate: Double,
    heirMarginalRate: Double
) -> HeirBracketComparison {
    let userTax = conversionAmount * userMarginalRate
    let heirTax = conversionAmount * heirMarginalRate
    return HeirBracketComparison(
        conversionAmount: conversionAmount,
        userMarginalRate: userMarginalRate,
        heirMarginalRate: heirMarginalRate,
        userTaxIfConvertedNow: userTax,
        heirTaxIfInheritedLater: heirTax,
        netFamilyBenefit: heirTax - userTax
    )
}
```

```swift
/// Estimated lifetime tax delta a survivor would pay above MFJ rates, summed over the
/// remaining single-filer years. Negative input = no delta.
func widowLifetimeTaxDelta(conversionAmount: Double, survivorYears: Int) -> Double {
    guard widowBracketApplies else { return 0 }
    let baseIncome = widowComparisonIncome
    let adjusted = max(0, baseIncome - conversionAmount)
    let mfjAnnual = TaxCalculationEngine.widowMFJAnnualTax(rmdIncome: adjusted)
    let singleAnnual = TaxCalculationEngine.widowSurvivorAnnualTax(rmdIncome: adjusted * 0.85)
    return (singleAnnual - mfjAnnual) * Double(max(0, survivorYears))
}
```
