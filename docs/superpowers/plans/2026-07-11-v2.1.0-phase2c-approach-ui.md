# V2.1.0 Phase 2c — Conversion-Approach UI (Multi-Year tab) — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface the three built conversion approaches in the Multi-Year tab: a "Conversion approach" control (segmented + config-derived inline bracket/tier chips with baseline-above-target status), the three-way comparison view (replacing today's two-way block), path-level consequence display, a light giving refinement, and propagation so the selected approach becomes the **active plan everywhere** (charts, balances, Year-1 levers, CPA briefing). No engine or Phase-2b-model changes.

**Architecture:** All heavy logic already exists in the engine (`OptimizationEngine.optimize(approach:)`) and the Phase-2b model (`ApproachComparison`/`ApproachColumn`/`ConsequenceDeltas`/`ConsequenceFlags`/`ApproachComparisonCoordinator`/`MultiYearCPABriefing.approachDeltaSummary`). This phase adds: (1) persistence of the selection in `MultiYearAssumptions`; (2) an async `computeApproachComparison()` on `MultiYearStrategyManager` that mirrors the existing `computeHeirFrontier()` exactly; (3) pure, unit-tested view-logic helpers (anchor label, baseline-target classification, active-path selection, Year-1 revert rule); (4) two SwiftUI views (`ApproachComparisonView`, `ConversionApproachSection`) verified via the browser-preview workflow; (5) integration wiring in `MultiYearPlanView` + the CPA briefing. Testable logic is extracted into pure functions so the SwiftUI itself carries no untested branching.

**Tech Stack:** Swift, SwiftUI, Swift `Testing`, `@testable import RetireSmartIRA`. Native macOS/iOS Xcode (new `.swift` files auto-compile via filesystem-synchronized project groups — no `pbxproj` edits).

## Global Constraints

- **No "Recommended" anywhere — the app does not make recommendations.** The anchor column and the picker's first segment are labeled by the objective the engine actually optimized, dynamically: **"Minimize lifetime tax"** when the anchor's effective heir weight is 0, **"Optimize tax + legacy"** when it is > 0. One helper produces this string for the picker segment, the comparison column header, and the CPA briefing so all three read identically.
- **Effective heir weight** (the legacy-on signal) = `dataManager.enableLegacyPlanning ? manager.selectedHeirWeight : 0`. The anchor's `optimize(heirWeight:)` and the label both key off this same value.
- **Label scheme (verbatim):** anchor/segment-1 = "Minimize lifetime tax" / "Optimize tax + legacy"; `.fillToBracket(rate)` column = "Fill to N% bracket"; `.limitToIRMAA(tier)` column = "Limit to IRMAA tier N"; third comparison column = "No added Roth conversions"; Year-1 override on the objective optimizer = existing off-plan / "Custom" badge.
- **Bracket/tier chips are config-derived**, never hardcoded (`configProvider.config(forYear:).toTaxBrackets()` for ordinary brackets; the config's IRMAA tiers). Show all supported brackets/tiers; a target already below the year's unavoidable baseline is labeled "not achievable this year" but the policy stays active for later years — never filtered solely from year one.
- **NIIT flag copy:** "NIIT increased because MAGI crossed the threshold and the household has net investment income" — never implying the Roth conversion itself is net investment income.
- **Year-1 mutual exclusion:** a deterministic approach and a manual Year-1 Roth override cannot both hold; editing Year-1 reverts the selection to `.recommendedTaxMin` (the objective optimizer) with the override applied.
- **Three-way replaces two-way:** `ApproachComparisonView` replaces `PlanComparisonView` in the tab. The `PlanComparison` model, its CPA-briefing use, and `TaxImpactChart` are untouched.
- **Additive / no rebaselines:** this phase is UI + manager wiring + persistence on top of the finished engine/model. Full suite (**~1,748 baseline**) stays green; no existing computed number changes.
- **Authoritative test counts:** `-resultBundlePath` + `xcresulttool` (xcodebuild stdout undercounts).

```swift
import Foundation

/// Codable persistence shim for `ConversionApproach` (which has associated values). Stored on
/// `MultiYearAssumptions` so the user's selected approach survives relaunch.
struct PersistedConversionApproach: Codable, Equatable, Sendable {
    enum Kind: String, Codable { case recommendedTaxMin, fillToBracket, limitToIRMAA }
    let kind: Kind
    let rate: Double?
    let tier: Int?
    let buffer: Double?

    static let recommendedTaxMin = PersistedConversionApproach(.recommendedTaxMin)

    init(_ approach: ConversionApproach) {
        switch approach {
        case .recommendedTaxMin:
            kind = .recommendedTaxMin; rate = nil; tier = nil; buffer = nil
        case .fillToBracket(let r):
            kind = .fillToBracket; rate = r; tier = nil; buffer = nil
        case .limitToIRMAA(let t, let b):
            kind = .limitToIRMAA; rate = nil; tier = t; buffer = b
        }
    }

    func toApproach() -> ConversionApproach {
        switch kind {
        case .recommendedTaxMin: return .recommendedTaxMin
        case .fillToBracket:     return .fillToBracket(rate: rate ?? 0.24)
        case .limitToIRMAA:      return .limitToIRMAA(tier: tier ?? 1, buffer: buffer ?? 5_000)
        }
    }
}
```

    /// Compute the three-way approach comparison off the main actor and publish it. Mirrors
    /// computeHeirFrontier's cadence: cold-start spinner, silent refresh thereafter.
    func computeApproachComparison() {
        guard let dataManager, let scenarioStateManager else { return }
        let assumptions = self.assumptions
        let configProvider = self.configProvider
        let approach = assumptions.conversionApproach.toApproach()
        let heirWeight = dataManager.enableLegacyPlanning ? selectedHeirWeight : 0
        let inputs = MultiYearInputAdapter.build(
            from: dataManager, scenarioState: scenarioStateManager,
            assumptions: assumptions, excludeYear1Overrides: false)
        if approachComparison == nil { isComputingComparison = true }
        comparisonWorkTask?.cancel()
        let work = Task.detached(priority: .utility) {
            ApproachComparisonCoordinator().compare(
                inputs: inputs, assumptions: assumptions,
                selectedApproach: approach, heirWeight: heirWeight,
                configProvider: configProvider)
        }
        comparisonWorkTask = work
        Task { @MainActor [weak self] in
            let result = await work.value
            guard let self, !Task.isCancelled, !work.isCancelled else { return }
            self.approachComparison = result
            if self.isComputingComparison { self.isComputingComparison = false }
        }
    }
```

import Foundation

/// Pure, view-agnostic logic for the conversion-approach UI. Keeps the SwiftUI branch-free and
/// unit-testable. No "Recommended" strings — the anchor is labeled by the objective the engine
/// actually optimized (legacy-off = lifetime-tax min; legacy-on = tax+legacy blend).
enum ApproachUILogic {
    static func anchorLabel(effectiveHeirWeight: Double) -> String {
        effectiveHeirWeight > 0 ? "Optimize tax + legacy" : "Minimize lifetime tax"
    }

    static func columnLabel(_ approach: ConversionApproach, effectiveHeirWeight: Double) -> String {
        switch approach {
        case .recommendedTaxMin:      return anchorLabel(effectiveHeirWeight: effectiveHeirWeight)
        case .fillToBracket(let r):   return "Fill to \(Int((r * 100).rounded()))% bracket"
        case .limitToIRMAA(let t, _): return "Limit to IRMAA tier \(t)"
        }
    }

    enum TargetStatus: Equatable { case reachable, exceededByBaseline, notApplicable }

    static func bracketStatus(bracketTopOrdinaryIncome: Double, baselineOrdinaryIncome: Double) -> TargetStatus {
        baselineOrdinaryIncome >= bracketTopOrdinaryIncome ? .exceededByBaseline : .reachable
    }

    static func activePath(selected: ConversionApproach,
                           comparison: ApproachComparison?,
                           frontierOrCurrent: [YearRecommendation]) -> [YearRecommendation] {
        switch selected {
        case .recommendedTaxMin: return frontierOrCurrent
        case .fillToBracket, .limitToIRMAA: return comparison?.selected.path ?? frontierOrCurrent
        }
    }

    static func approachAfterYear1Edit(_ selected: ConversionApproach) -> ConversionApproach {
        .recommendedTaxMin
    }
}
```
