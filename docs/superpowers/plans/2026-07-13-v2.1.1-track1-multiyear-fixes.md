# V2.1.1 Track 1 — Multi-Year reporting/correctness fixes

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Fix the confirmed Multi-Year reporting/correctness bugs (B4 phantom conversions, A3 gross-up-excluded-from-MAGI, A4 gross-up disclosure, B2 nominal-vs-PV lifetime tax) so displayed figures match the executed plan. Track 2 (A1/A5 optimizer) is a separate effort.

**Architecture:** The projection already clamps conversions to the available balance, so `taxBreakdown`/balances are correct; the bugs are (a) the ladder REQUESTS conversions from a drained IRA and (b) six reporting sites read the requested amount instead of the executed one, plus (c) the gross-up withdrawal is excluded from forward IRMAA/NIIT MAGI and (d) the comparison shows nominal not PV lifetime tax. Fix the request cap, add an `executedRothConversion` field as the single source of truth, route all consumers to it, correct the stored MAGI, and show PV.

**Tech Stack:** Swift, Swift Testing, Xcode (macOS destination). Branch `2.1.1/multiyear-fixes` off `main` (which is now 2.1.0 / build 60).
